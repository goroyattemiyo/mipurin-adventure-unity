# WebApp版シナリオ参考メモ

Unity版『ミプリンの冒険』をRPG化するにあたり、WebApp版 `goroyattemiyo/mipurin-adventure` に存在するシナリオ・NPC・世界観・ショップ・図鑑要素を抽出し、Unity版で再構築するための参考メモとして整理する。

この文書は、WebApp版をそのまま移植するためのものではない。
Unity版では、現在の美麗2D・村拠点型アクションRPG方針に合わせて、設定を整理・再設計する。

## 参照元

WebApp版参考元リポジトリ：

```text
goroyattemiyo/mipurin-adventure
```

## WebApp版から分かる重要方針

WebApp版には、すでに以下の要素が存在する。

```text
プロローグ
NPC会話
ショップ
アイテム名
ボス会話
エンディング分岐
世界ロア
図鑑
花壇強化
祝福
武器
メタ進行
```

そのため、Unity版のRPG化は新規追加というより、WebApp版で育っていた要素をUnity版の構造に合わせて再構築する流れとして扱う。

## Unity版で採用したい最終方向性

```text
村拠点型・かわいい蜂ファンタジー・アクションRPG
```

基本ループ：

```text
村でNPCと話す
↓
ショップ・装備・図鑑・強化を確認する
↓
冒険エリアへ出発する
↓
部屋制アクション戦闘を行う
↓
素材・はちみつ・装備・ロアを入手する
↓
村へ戻る
↓
会話・商品・解放要素が変化する
↓
次の章へ進む
```

## プロローグ要素

WebApp版 `lang/ja.json` には、以下のような導入がある。

```text
ハニービレッジの蜂蜜は、やさしい記憶から生まれる
女王が黄金蜂蜜を守っていた
ある日、女王が消え、黄金蜂蜜は三つに砕ける
世界は少しずつ色を失う
ミプリンが忘れものを取り戻しに行く
```

Unity版では、この要素を軸にする。

### Unity版での整理案

```text
ハニービレッジでは、蜂蜜はただの食べ物ではなく、記憶と心から生まれる特別な力だった。
その中心には、花の国と蜂の王国をつなぐ黄金蜂蜜があった。

しかし、女王レイラが姿を消した日、黄金蜂蜜は三つに砕け、世界の色は少しずつ失われていく。

小さな冒険者みぷりんは、針型スピアとはちみつポットを手に、失われた記憶と女王の行方を探す旅に出る。
```

## 表記・設定の揺れ

WebApp版には、今後Unity版で整理した方がよい表記揺れがある。

```text
ハニービレッジ / ハニーヴィル
女王レイラ / 女王フローラ / フローラ
黄金蜂蜜 / クリスタル
花の国 / 蜂の王国
```

Unity版では、まず以下の仮整理で進める。

```text
村名：ハニービレッジ
女王：女王レイラ
支援NPC：フローラ
中心アイテム：黄金蜂蜜
力の断片：黄金蜂蜜のかけら
災厄：闇胞子
```

ただし、フローラを女王名として使うか、支援NPC名として使うかは要検討。
WebApp版のロアとエンディングで役割が混ざっている可能性があるため、Unity版では早めに固定する。

## NPC会話分岐の方向性

WebApp版では、会話が以下の状態で変化する。

```text
killCount
needleUseCount
黄金蜂蜜のかけら所持
エンディング到達
イベントフラグ
```

Unity版では、まず複雑な分岐を入れず、段階式にする。

### 初期段階

```text
未出発
初回冒険後
かけら入手後
第1章クリア後
```

### 将来段階

```text
敵を倒しすぎた場合
針の力を使いすぎた場合
図鑑を埋めた場合
特定ボスを倒した場合
特定アイテムを持っている場合
```

行動で村人の反応が変わる仕組みは、Unity版でも採用価値が高い。
ただし、初期実装では会話条件を少なくし、イベント管理が破綻しないようにする。

## 重要シナリオ要素

### 女王失踪

WebApp版では、女王が消えたことで黄金蜂蜜が砕け、世界が色を失っていく導入がある。
Unity版でも第1章の大きな背景にする。

### 黄金蜂蜜のかけら

WebApp版には、黄金蜂蜜のかけらA/B/Cが存在する。
Unity版でも章ごとの目標として使いやすい。

```text
第1章：かけらA
第2章：かけらB
第3章：かけらC
```

### ミプリンの両親

WebApp版では、ミプリンの父タイガ、母ハナに関する伏線がある。

抽出できる要素：

```text
母ハナは商人マルシェと昔一緒に冒険していた
母の蜂蜜ポットが残されている
父タイガは森の切株に鍵を隠していた
両親は10年前、女王を探して奥地へ進み、戻らなかった
```

Unity版では、この両親の過去を中盤以降の感情軸にする。

### 記憶と蜂蜜

WebApp版では、蜂蜜は記憶と結びついている。
これはUnity版の独自性として強い。

採用案：

```text
蜂蜜は、心と記憶を映す。
やさしい記憶からは黄金蜂蜜が生まれる。
傷ついた記憶や闇胞子に触れた蜂蜜は黒く濁る。
```

この設定により、図鑑・会話・敵・アイテムがつながる。

## Unity版で整理・変更するもの

### 1. 用語統一

```text
ハニービレッジ
女王レイラ
黄金蜂蜜
黄金蜂蜜のかけら
闇胞子
フローラは支援NPC候補
```

### 2. 戦闘との接続

WebApp版はローグライト色が強い。
Unity版では、以下のように接続する。

```text
VillageTest
- 村、会話、ショップ

CombatRoom / CharacterTest
- 部屋制戦闘

StoryProgress
- 戦闘結果で村の状態と会話が変化
```

### 3. シナリオ密度

Unity版では、美麗2Dキャラと会話演出が強みになる。
そのため、長文ロアよりも、短い会話と小さなイベントで見せる。

```text
1会話：2〜4行
重要イベント：3〜6ページ
図鑑ロア：短文
```

## 最初のRPG MVP

いきなり本格RPGにしない。
まずは以下を作る。

```text
VillageTestシーンを作る
NPCを1人置く
近づいて話せる
2〜3行の会話を表示する
村からCharacterTestへ移動する
CharacterTestクリア後に村へ戻る
NPCの会話が1段階変わる
```

これができれば、Unity版のRPG化は成立する。

## 判断

WebApp版には、Unity版RPG化に使える材料が十分ある。

Unity版では、WebApp版の全部を移植するのではなく、以下の考えで使う。

```text
WebApp版 = システムとシナリオ素材の参考元
Unity版 = 美麗2Dで再構築する正本
```

現時点では、Unity版の方向性を以下に更新する。

```text
Unity版『ミプリンの冒険』は、
部屋制アクション戦闘を土台にしつつ、
村拠点、NPC会話、ショップ、アイテム、図鑑、装備、シナリオ進行を持つ
かわいい蜂ファンタジー・アクションRPGとして発展させる。
```
