# 10 Collection and Cosmetic System

## 目的

Unity版『ミプリンの冒険』における、装備・アイテム・図鑑・着せ替え・コレクション要素の将来方針をまとめる。

これは今すぐ実装するものではなく、最終系のやり込み要素として設計に残す。

## 採用判断

```text
採用：あり
優先度：中〜高
実装時期：CharacterTest / 1部屋ゲーム / ローグライト基礎の後
位置づけ：やり込み・コレクション・見た目カスタム要素
```

みぷりんはキャラクターの可愛さが強みなので、装備・アイテム・図鑑・着せ替えはゲームの差別化ポイントになる。

## 基本方針

最終的には以下を入れる。

```text
かわいい装備を集める
↓
図鑑に登録される
↓
見た目を変えられる
↓
性能装備と見た目装備を分ける
```

## 装備と見た目を分ける

性能と見た目を同じにすると、以下の問題が起きる。

```text
強いけど見た目が好きではない
好きな見た目だけど性能が弱い
```

そのため、最終的には以下の2系統に分ける。

```text
装備：ゲーム性能に影響する
コスメ：見た目だけ変わる
```

## 装備アイテム

ゲーム中の性能に関わるもの。

例：

```text
針スピア
はちみつ杖
花粉ショット
羽ばたきブーツ
ミツロウシールド
```

効果例：

```text
攻撃力アップ
攻撃速度アップ
射程アップ
移動速度アップ
被ダメージ軽減
特殊攻撃追加
```

## 見た目アイテム / コスメ

性能とは別に、みぷりんの外見を変えるもの。

例：

```text
花かんむり
蜂王国のリボン
探検バッグ
星形ヘアピン
小さなマント
みつばち騎士帽
羽色違い
足元エフェクト
```

## 最初に実装しやすい着せ替え

現在のアニメーション方式は、

```text
本体フレームアニメ + 羽だけ別レイヤー + エフェクト
```

なので、本格的な衣装差分をすぐ作ると素材数が爆発する。

そのため、最初は本体フレームに影響しにくい小物から始める。

優先候補：

```text
頭アクセ
背中アクセ
手持ちアクセ
足元エフェクト
羽色違い
```

## 後回しにする着せ替え

以下は素材負荷が高いので後回し。

```text
全身衣装差分
髪型変更
体型変更
全方向アニメ対応衣装
攻撃フレームごとの衣装差分
```

## 図鑑

図鑑は重要なやり込み要素として採用する。

候補：

```text
敵図鑑
武器図鑑
祝福図鑑
装備図鑑
衣装図鑑
素材図鑑
```

## 図鑑テキスト方針

図鑑には、短い説明文を入れる。

例：

```text
はちみつスライム
甘い香りにつられて集まるぷるぷるの魔物。
倒すと、ときどき濃いはちみつを落とす。
```

## コレクションルーム

将来的に、集めた装備・衣装・図鑑を確認できる場所を作る。

場所候補：

```text
みぷりんの家
花壇
はちみつ工房
衣装棚
図鑑部屋
```

## 実装順

### Phase 1

- 敵図鑑の土台
- 敵を倒したら登録
- 図鑑に名前だけ表示

### Phase 2

- 武器・祝福図鑑
- 入手済み/未入手表示
- 短い説明文

### Phase 3

- 装備ドロップ
- 装備一覧
- 装備効果

### Phase 4

- コスメ入手
- 拠点で見た目変更
- 小物アクセから対応

### Phase 5

- コレクションルーム
- 衣装棚
- 図鑑コンプリート報酬

## 今はやらないこと

- 全身衣装差分
- 複雑な着せ替えUI
- 課金的なスキン管理
- 大量の装備実装
- 図鑑コンプリート報酬

まずはゲーム本体の1部屋ローグライトが成立してから進める。

## 判断

装備・アイテム・図鑑・着せ替えは、みぷりんUnity版の将来要素として採用する。

ただし、今すぐ作るのではなく、ゲームの基本ループが成立した後に段階的に入れる。
